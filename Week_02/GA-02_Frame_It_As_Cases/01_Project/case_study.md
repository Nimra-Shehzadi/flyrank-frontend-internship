# Settings Form Case Study

## Problem

Many forms collect user information but provide poor validation and unclear feedback. I wanted to build a settings form that gives users a smooth experience with proper validation and clear success and error messages.

## What I Did

I built a responsive Settings Form using HTML, CSS, and JavaScript. The form includes Full Name, Email Address, Theme selection, and a Notifications checkbox. I implemented validation for required fields and email format, displayed meaningful error messages, and showed a success message after successful submission.

## Result

The project helped me improve my frontend development skills and practice AI-assisted development. I also learned how better prompts produce better code and documentation while keeping the final implementation understandable and maintainable.
