Problem
A lot of forms collect user info but do a poor job of validating it or telling the user what went wrong. I wanted to build a settings form for FlyRank that actually gets this right — clear validation, clear error messages, and a clear success state, so users aren't left guessing.
What I Did
I built a responsive Settings Form using HTML, CSS, and JavaScript. It collects Full Name, Email Address, a Theme selection, and a Notifications checkbox. I added validation for required fields and email format, and made sure error and success messages were specific and easy to understand. I also used AI tools during development, and part of the work was learning how to write better prompts to get cleaner, more usable code and documentation — while still making sure I understood and could maintain everything myself.
Result
The form works as intended for a real, production use case at FlyRank. Beyond the form itself, I got better at frontend development and at working with AI tools — specifically, learning that a vague prompt gets you a vague answer, and a specific prompt gets you something you can actually use.
