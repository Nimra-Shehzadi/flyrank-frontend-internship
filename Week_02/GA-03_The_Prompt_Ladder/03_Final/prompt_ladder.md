# Prompt Ladder

## Baseline Prompt
Created a basic prompt with only the core requirements for a responsive Settings Form.

### Improvement
The response was functional but lacked context and detailed guidance.

---

## Version 1 – Audience
### Change Made
Added the target audience (beginner frontend developers).

### Improvement
The AI generated simpler and easier-to-understand code suitable for beginners.

---

## Version 2 – Constraints
### Change Made
Specified that only HTML, CSS, and JavaScript should be used.

### Improvement
The AI avoided frameworks and produced cleaner, beginner-friendly code.

---

## Version 3 – Output Format
### Change Made
Specified that the response should be separated into `index.html`, `style.css`, and `script.js`.

### Improvement
The output became better organized and easier to use.

---

## Version 4 – Quality Criteria
### Change Made
Added requirements for semantic HTML, responsiveness, accessibility, and readable code.

### Improvement
The generated solution became more professional and maintainable.

---

## Version 5 – Verification
### Change Made
Asked the AI to verify every requirement before finishing and provide a checklist.

### Improvement
The response became more complete and easier to review.

---

# Key Learning

Small improvements to prompts produce significantly better AI responses. Adding audience, constraints, output format, quality criteria, and verification makes AI-generated code more accurate, organized, and reliable.
